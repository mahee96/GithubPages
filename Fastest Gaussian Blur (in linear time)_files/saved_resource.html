                                                          Marc Prez a year ago

                                                          Very interesting!
                                                          To whoever may concern, I did the 4 channels trick. here it is:

                                                          let rData: Uint8ClampedArray = new Uint8ClampedArray(d.data.length / 4);
                                                          let gData: Uint8ClampedArray = new Uint8ClampedArray(d.data.length / 4);
                                                          let bData: Uint8ClampedArray = new Uint8ClampedArray(d.data.length / 4);

                                                          for (let i = 0; i < d.data.length; i = i + 4) {
                                                          rData[i / 4] = (d.data[i]);
                                                          gData[i / 4] = (d.data[i + 1]);
                                                          bData[i / 4] = (d.data[i + 2]);
                                                          }

                                                          gaussBlur(rData, rData, w, h, 3);
                                                          gaussBlur(gData, gData, w, h, 3);
                                                          gaussBlur(bData, bData, w, h, 3);

                                                          for (let i = 0; i < d.data.length; i = i + 4) {
                                                          d.data[i] = rData[i / 4]
                                                          d.data[i + 1] = gData[i / 4]
                                                          d.data[i + 2] = bData[i / 4]
                                                          d.data[i + 3] = 255
                                                          }
                                                          ctx.putImageData(d, 0, 0);

                                                                  Puscasu Robert a year ago

                                                                  From what i understand, Algorithm 4 is only good if you use it on the CPU once. On a GPU, for game development, you can use parallel programming to get the following algorithm that is faster than this one:

                                                                  For each pixel in the image, calculate the sum of all his neighbours multiplied by the Gaussian weights of each neighbour. The number of neighbours a pixel has is the sample size. KernelSize= 2*sampleSize +1 . This of course can be optimised by running a horizontal and vertical blur like you explained in this tutorial. Since the GPU runs a thread for each fragment, the complexity of this algorithm on a GPU is : KernelSize*2. One horizontal pass takes KernelSize operations and one vertical pass takes KernelSize operations.

                                                                  If i said anything wrong in this post, please correct me. For the past week ive been trying to find the fastest blur algorithm for game development.

                                                                                                                                              Hi, your raster image is usually stored inside some array, which is called a channel. When your image is 4 pixels wide and 3 pixels tall, e.g.

                                                                                                                                              92 3 45 8

Images often have three channels: Red, Green, Blue, so you should blur each channel separately.
